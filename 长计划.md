## 1. 现状分析

### 1.1 当前 AI 架构（问题）

```
前端 ──WebSocket──> api/room.py ──> core/llm_service.py ──httpx──> DeepSeek/Ollama API
                                          │
                                    core/prompts.py  (硬编码 prompt 模板)
                                          │
                                    core/agent.py    (AI 主持人，无工具、无记忆链)
                                    core/ai_player.py (AI 玩家，仅 Redis 短期记忆)
```

**核心痛点**:

| # | 问题 | 影响 |
|---|------|------|
| 1 | `llm_service.py` 直接用 httpx 拼请求 | 切换模型/供应商需改代码，无法利用 LangChain 生态 |
| 2 | Prompt 全部硬编码在 `prompts.py` | 无法动态组装上下文，prompt 膨胀后维护困难 |
| 3 | AI 主持人 (`agent.py`) 无工具调用能力 | 不能查题库、不能分析玩家历史、不能自主决策 |
| 4 | AI 玩家 (`ai_player.py`) 记忆仅靠 Redis KV | 无语义检索，无法回忆"类似局面"的策略经验 |
| 5 | 题库 (`puzzle_bank`) 仅支持随机抽取 | 无法按语义相似度找"与当前话题相关"的题目 |
| 6 | 无 Chain 编排 | 复杂推理（如评估玩家答案）靠单次 LLM 调用，准确率低 |

### 1.2 现有代码中可复用的部分

| 模块 | 可复用度 | 说明 |
|------|---------|------|
| `core/config.py` | ✅ 全部 | 新增 LangChain/向量库配置即可 |
| `core/db.py` | ✅ 全部 | SQLAlchemy 引擎通用 |
| `core/security.py` | ✅ 全部 | JWT 逻辑不变 |
| `core/state_machine.py` | ✅ 全部 | FSM 与 Agent 无关 |
| `core/task_queue.py` | ✅ 全部 | 异步任务队列不变 |
| `core/json_parser.py` | ✅ 全部 | JSON 解析工具可复用 |
| `core/undercover.py` | 🔄 部分 | 游戏规则引擎保留，AI 调用部分替换为 Agent |
| `core/redis_service.py` | 🔄 部分 | 保留缓存层，新增向量检索缓存 |
| `core/llm_service.py` | ❌ 替换 | 被 LangChain LLM 封装替代 |
| `core/prompts.py` | ❌ 替换 | 被 LangChain PromptTemplate + RAG 上下文替代 |
| `core/agent.py` | ❌ 替换 | 被 LangChain Agent 替代 |
| `core/ai_player.py` | ❌ 替换 | 被 LangChain Agent + Tool 替代 |

## 2. 改造目标

### 2.1 功能目标

1. **Agent 化 AI 主持人**: 拥有工具（查题库、查玩家历史、生成提示）、记忆（对话缓冲 + 向量长期记忆）、推理链的智能主持人
2. **RAG 增强题库**: 将题库向量化，支持语义检索（"给我一个和'密室'相关的推理题"）
3. **Agent 化 AI 玩家**: 卧底游戏中 AI 玩家拥有策略工具（分析投票历史、检测矛盾、模拟对手思维）
4. **Chain 编排**: 将"生成题目 → 审核质量 → 存入题库"等复杂流程编排为 Chain
5. **可扩展架构**: 新增游戏类型只需新增 Tool + Chain，无需重写底层

### 2.2 非目标（本次不做）

- ❌ 不改前端（保持现有 WebSocket 协议不变）
- ❌ 不引入流式输出（本次改造范围）
- ❌ 不做多模态（图片/语音）
- ❌ 不引入外部 Agent 框架（如 AutoGen、CrewAI），仅用 LangChain 原生 Agent

## 3. 技术选型

### 3.1 核心依赖

| 组件 | 选型 | 用途 |
|------|------|------|
| Agent 框架 | **LangChain** (`langchain-core`, `langchain`, `langchain-community`) | Agent、Chain、Tool、Memory 编排 |
| LLM 封装 | **`langchain-deepseek`** / **`langchain-ollama`** | 替代手写 httpx 调用 |
| 向量数据库 | **PGVector** (`pgvector` + `langchain-postgres`) | 复用现有 PostgreSQL，无需额外服务 |
| Embedding | **`langchain-huggingface`** (BAAI/bge-small-zh-v1.5) | 中文文本向量化，本地运行无额外 API |
| 文档加载 | **`langchain-text-splitters`** | 题库文本分块 |
| 输出解析 | **`langchain-core`** built-in | 结构化 JSON 输出 |

### 3.2 为什么选 PGVector 而不是独立向量库

| 方案 | 优点 | 缺点 |
|------|------|------|
| **PGVector (选定)** | 复用已有 PostgreSQL，零运维增量，SQL 可直接 JOIN 用户/题库表 | 性能略低于专用库（万级数据够用） |
| Milvus | 性能强 | 需要额外部署，运维成本高 |
| FAISS | 纯内存，快 | 不持久化，重启丢失 |
| Chroma | 简单 | 不适合生产，与 PG 数据无法 JOIN |

> 本项目题库上限 50 条、聊天记录万级规模，PGVector 完全够用。

### 3.3 Embedding 模型选型

| 模型 | 维度 | 大小 | 说明 |
|------|------|------|------|
| **BAAI/bge-small-zh-v1.5 (选定)** | 512 | ~90MB | 中文效果好，CPU 可跑，HuggingFace 免费 |
| m3e-base | 768 | ~400MB | 中文也不错，但更大 |
| OpenAI text-embedding-3-small | 1536 | 云端 | 需付费 API，有延迟 |

> 选择本地模型避免额外 API 依赖和费用。

### 5.4 新建 `core/llm_factory.py` — LLM 工厂

替代 `core/llm_service.py`，使用 LangChain LLM 封装：

```python
"""LLM 工厂：统一创建 LangChain LLM 实例。"""

from langchain_deepseek import ChatDeepSeek
from langchain_ollama import ChatOllama
from core.config import (
    LLM_MODE, LLM_API_KEY, LLM_API_URL, LLM_API_MODEL,
    OLLAMA_HOST, OLLAMA_MODEL, LLM_TEMPERATURE, LLM_MAX_TOKENS
)

def create_llm(mode: str = None, **kwargs):
    """根据配置创建 LangChain LLM 实例。

    Args:
        mode: "external" 或 "local"，默认读环境变量
        kwargs: 覆盖默认参数（temperature, max_tokens 等）
    """
    mode = mode or LLM_MODE
    defaults = {
        "temperature": kwargs.get("temperature", LLM_TEMPERATURE),
        "max_tokens": kwargs.get("max_tokens", LLM_MAX_TOKENS),
    }

    if mode == "external":
        return ChatDeepSeek(
            model=LLM_API_MODEL,
            api_key=LLM_API_KEY,
            base_url=LLM_API_URL,
            **defaults,
        )
    elif mode == "local":
        return ChatOllama(
            model=OLLAMA_MODEL,
            base_url=OLLAMA_HOST,
            **defaults,
        )
    else:
        raise ValueError(f"不支持的 LLM 模式: {mode}")


def create_llm_with_fallback(**kwargs):
    """创建带降级的 LLM：external → local → puzzle_bank。"""
    # 主 LLM
    primary = create_llm(**kwargs)
    # 备用 LLM
    fallback_mode = "local" if LLM_MODE == "external" else "external"
    try:
        fallback = create_llm(mode=fallback_mode, **kwargs)
        return primary.with_fallbacks([fallback])
    except Exception:
        return primary
```

**关键改动**:
- `llm_service.chat(prompt, system_prompt)` → LangChain 的 `llm.invoke(messages)` 格式
- 支持 `with_fallbacks` 实现降级链
- 返回 LangChain LLM 对象，可直接用于 Chain / Agent

```python
from langchain_huggingface import HuggingFaceEmbeddings
from core.config import EMBEDDING_MODEL, EMBEDDING_DEVICE

_embedding_instance = None

def get_embedding():
    """获取全局单例 Embedding 模型（懒加载）。"""

            encode_kwargs={"normalize_embeddings": True},

    return _embedding_instance
```

### 5.6 新建 `core/vector_store.py` — 向量库管理

```python
"""PGVector 向量库管理。"""

from langchain_postgres import PGVector
from core.config import DATABASE_URL, VECTOR_COLLECTION_PREFIX
from core.embedding import get_embedding

def get_vector_store(collection_name: str) -> PGVector:
    """获取指定集合的 PGVector 存储。

    Args:
        collection_name: 集合名（自动加前缀），如 "puzzles", "chat_history"

    return PGVector(
        embeddings=get_embedding(),
        collection_name=f"{VECTOR_COLLECTION_PREFIX}{collection_name}",
        connection=DATABASE_URL.replace("+psycopg2", "+psycopg"),  # langchain-postgres 用 psycopg3
        use_jsonb=True,
```

### 6.1 RAG 数据源分析

| 数据源 | 当前存储 | 向量化内容 | 检索场景 |

### 6.2 新建 `core/rag/splitter.py` — 文本分块

```python
"""文本分块策略：针对不同数据源使用不同分块方式。"""

from langchain_text_splitters import RecursiveCharacterTextSplitter

def puzzle_splitter():
    """题库分块：每道题是一个完整块，不需要切分。"""
    return RecursiveCharacterTextSplitter(
        chunk_size=1000,       # 题目通常 < 500 字
        chunk_overlap=0,       # 题目之间无关联，不需要重叠
        separators=["\n\n", "\n", "。", "，"],
    )

def chat_history_splitter():
    """聊天记录分块：按轮次分块，保留上下文。"""
    return RecursiveCharacterTextSplitter(
        chunk_size=500,
        chunk_overlap=100,     # 保留前后各一轮的上下文
        separators=["\n", "。"],
    )

def rule_text_splitter():
    """规则文本分块：按章节分块。"""
    return RecursiveCharacterTextSplitter(
        chunk_size=800,
        chunk_overlap=100,
        separators=["\n\n", "\n", "。"],
    )
```

### 6.3 新建 `core/rag/ingester.py` — 数据灌入

```python
"""数据灌入：将结构化数据转为向量存入 PGVector。"""

from langchain_core.documents import Document
from core.vector_store import get_vector_store
from core.db import SessionLocal
from models.db_models import PuzzleBank, Message

def ingest_puzzles():
    """将 puzzle_bank 表中所有题目灌入向量库。"""
    vs = get_vector_store("puzzles")
    session = SessionLocal()
    try:
        puzzles = session.query(PuzzleBank).all()
        docs = []
        for p in puzzles:
```

### 6.4 新建 `core/rag/retriever.py` — 检索器

```python
"""RAG 检索器：统一接口，按场景检索相关知识。"""

from typing import Optional
from core.vector_store import get_vector_store
from core.config import VECTOR_SEARCH_TOP_K, VECTOR_DISTANCE_THRESHOLD

class RAGRetriever:
    """统一 RAG 检索器。"""

    def retrieve_puzzles(self, query: str, k: int = None, category: str = None) -> list:
        """语义检索题库。

        Args:
            query: 检索语义，如 "密室杀人 推理"
            k: 返回数量，默认 VECTOR_SEARCH_TOP_K
            category: 可选分类过滤
        """
        vs = get_vector_store("puzzles")
        k = k or VECTOR_SEARCH_TOP_K
        results = vs.similarity_search_with_score(query, k=k)
        if category:
            results = [(doc, score) for doc, score in results
                       if doc.metadata.get("category") == category]
        return [
            {"content": doc.page_content, "metadata": doc.metadata, "score": score}
            for doc, score in results
            if score >= VECTOR_DISTANCE_THRESHOLD
        ]

    def retrieve_chat_context(self, query: str, room_id: str = None, k: int = None) -> list:
        """检索相关聊天记录。"""
        vs = get_vector_store("chat_history")
        k = k or VECTOR_SEARCH_TOP_K
        filter_dict = {"room_id": room_id} if room_id else None
        results = vs.similarity_search(query, k=k, filter=filter_dict)
        return [doc.page_content for doc in results]

    def retrieve_rules(self, query: str, game_type: str = None, k: int = 3) -> list:
        """检索游戏规则。"""
        vs = get_vector_store("game_rules")
        filter_dict = {"game_type": game_type} if game_type else None
        results = vs.similarity_search(query, k=k, filter=filter_dict)
        return [doc.page_content for doc in results]
```

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import JsonOutputParser
from langchain_core.runnables import RunnableLambda, RunnablePassthrough

# Step 1: 生成 prompt（带 RAG 上下文）
gen_prompt = ChatPromptTemplate.from_template("""
你是一位海龟汤谜题出题人。根据以下参考题目风格，创作一道新的海龟汤。

分类偏好: {category}

参考题目（相似题目）:
{rag_context}

请严格输出以下 JSON 格式（不要输出其他内容）:
{{
    "title": "题目标题（10字以内）",
    "situation": "汤面描述（50-200字）",
    "truth": "汤底真相（50-300字）",
    "hints": ["提示1", "提示2", "提示3"],
    "category": "分类"
}}
""")

# Step 2: 质量审核 prompt
review_prompt = ChatPromptTemplate.from_template("""
你是一位海龟汤谜题审核员。请评估以下谜题的质量。

题目:
{puzzle_json}

评分标准：
1. 逻辑性（汤底是否合理解释汤面）: 0-10
2. 趣味性（是否有悬念和反转）: 0-10
3. 难度匹配（是否符合指定难度）: 0-10
4. 提示有效性（提示是否有助于推理）: 0-10

输出 JSON:
{{
    "scores": {{"logic": N, "fun": N, "difficulty": N, "hints": N}},
    "total": N,
    "pass": true/false,
    "suggestions": "改进建议（如果 pass=false）"
}}
""")

def build_puzzle_gen_chain(llm):
    """构建题目生成 Chain。

    流程: RAG 检索参考题 → LLM 生成 → LLM 审核 → 通过则入库
    """
    parser = JsonOutputParser()

    def get_rag_context(inputs):
        from core.rag.retriever import retriever
        query = f"{inputs['difficulty']}难度 {inputs.get('category', '')}海龟汤"
        results = retriever.retrieve_puzzles(query, k=3)
        return "\n---\n".join(r["content"] for r in results)

    gen_chain = (
        RunnableLambda(get_rag_context)
        | RunnablePassthrough.assign(rag_context=lambda x: x)
        | gen_prompt
        | llm
        | parser
    )

    review_chain = review_prompt | llm | parser

    # 完整流程：生成 → 审核
    return gen_chain | RunnablePassthrough.assign(
        review=lambda x: review_chain.invoke({"puzzle_json": str(x)})
    )
```

### 7.4 Agent 基类设计 (`core/agents/base_agent.py`)

```python
"""Agent 基类：统一工具注册、记忆管理、LLM 创建。"""

from abc import ABC
from langchain.agents import AgentExecutor, create_tool_calling_agent
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.tools import BaseTool

class BaseGameAgent(ABC):
    """游戏 Agent 基类。"""

    def __init__(self, llm=None, tools: list[BaseTool] = None, memory=None):
        self.llm = llm or create_llm()
        self.tools = tools or []
        self.memory = memory
        self._executor = None

    def build_prompt(self, system_prompt: str) -> ChatPromptTemplate:
        """构建 Agent 提示模板。"""
        return ChatPromptTemplate.from_messages([
            ("system", system_prompt),
            MessagesPlaceholder(variable_name="chat_history"),
            ("human", "{input}"),
            MessagesPlaceholder(variable_name="agent_scratchpad"),
        ])

    def get_executor(self, system_prompt: str) -> AgentExecutor:
        """获取 Agent 执行器（懒构建）。"""
        if self._executor is None:
            prompt = self.build_prompt(system_prompt)
            agent = create_tool_calling_agent(self.llm, self.tools, prompt)
            self._executor = AgentExecutor(
                agent=agent,
                tools=self.tools,
                memory=self.memory,
                verbose=True,
                max_iterations=5,
                handle_parsing_errors=True,
            )
        return self._executor

    async def run(self, input_text: str, system_prompt: str = "", **kwargs) -> str:
        """运行 Agent。"""
        executor = self.get_executor(system_prompt)
        result = await executor.ainvoke({"input": input_text, **kwargs})
        return result["output"]
```

```python
        return await self.run(user_message, system_prompt=system)

    async def generate_puzzle(self, difficulty: str) -> dict:
        """使用 Chain 生成题目。"""
        from core.chains.puzzle_gen import build_puzzle_gen_chain
        chain = build_puzzle_gen_chain(self.llm)
        result = await chain.ainvoke({"difficulty": difficulty, "category": ""})
        # 审核通过才入库
        if result.get("review", {}).get("pass", False):
            from core.puzzle_bank import puzzle_bank
            puzzle_bank.add(
                title=result["title"],
                situation=result["situation"],
                truth=result["truth"],
                category=result.get("category", "未分类"),
                hints=json.dumps(result.get("hints", []), ensure_ascii=False),
            )
        return result
```

### 8.2 裁判 Agent 改造 (`core/agents/judge_agent.py`)

**当前**: `core/runtime.py` 中的 task handler 直接调用 `llm_service.chat()` + `prompts.py`
**改造后**: 使用 Chain 编排

```python
class JudgeAgent:
    """裁判 Agent：负责判断提问和评估答案。"""

    def __init__(self, llm=None):
        self.llm = llm or create_llm()
        self._judge_chain = None
        self._eval_chain = None

    @property
    def judge_chain(self):
        if self._judge_chain is None:
            from core.chains.question_judge import build_judge_chain
            self._judge_chain = build_judge_chain(self.llm)
        return self._judge_chain

    @property
    def eval_chain(self):
        if self._eval_chain is None:
            from core.chains.answer_eval import build_answer_eval_chain
            self._eval_chain = build_answer_eval_chain(self.llm)
        return self._eval_chain

    async def judge_question(self, question: str, situation: str,
                              truth: str, history: list) -> dict:
        """判断玩家提问。"""
        history_text = "\n".join(f"Q: {h['q']} → A: {h['a']}" for h in history)
        return await self.judge_chain.ainvoke({
            "question": question,
            "situation": situation,
            "truth": truth,
            "history": history_text,
        })

    async def evaluate_answer(self, answer: str, situation: str, truth: str) -> dict:
        """评估玩家答案。"""
        return await self.eval_chain.ainvoke({
            "answer": answer,
            "situation": situation,
            "truth": truth,
        })
```

### 8.5 改造 `core/llm_service.py` — 保留为兼容层

不直接删除 `llm_service.py`，而是将其改造为 LangChain 的薄封装：

```python
"""LLM 服务兼容层：保留旧接口，内部调用 LangChain。"""

from core.llm_factory import create_llm

class LLMService:
    """向后兼容的 LLM 服务。"""

    def __init__(self):
        self._llm = None

    @property
    def llm(self):
        if self._llm is None:
            self._llm = create_llm()
        return self._llm

    async def chat(self, prompt: str, system_prompt: str = "",
                    temperature: float = None, max_tokens: int = None) -> str:
        """兼容旧接口：prompt → LangChain messages → invoke。"""
        from langchain_core.messages import HumanMessage, SystemMessage
        messages = []
        if system_prompt:
            messages.append(SystemMessage(content=system_prompt))
        messages.append(HumanMessage(content=prompt))

        kwargs = {}
        if temperature is not None:
            kwargs["temperature"] = temperature
        if max_tokens is not None:
            kwargs["max_tokens"] = max_tokens

        llm = create_llm(**kwargs) if kwargs else self.llm
        response = await llm.ainvoke(messages)
        return response.content

    def is_available(self) -> bool:
        try:
            return self.llm is not None
        except Exception:
            return False

# 全局单例（保持兼容）
llm_service = LLMService()
```

## 12. 风险与对策

| # | 风险 | 概率 | 影响 | 对策 |
|---|------|------|------|------|
| 1 | PGVector 在 Windows 上安装困难 | 中 | 无法使用向量库 | 备选方案：FAISS（纯 Python，无需编译），数据不持久化但可每次启动时重新灌入 |
| 2 | Embedding 模型首次下载慢（~90MB） | 高 | 首次启动慢 | 预下载到本地缓存，文档中说明 |
| 3 | LangChain 版本迭代快，API 变化大 | 中 | 升级困难 | 锁定版本号，核心接口做封装隔离 |
| 4 | Agent 调用工具导致响应变慢 | 中 | 用户体验下降 | 保留 task_queue 异步调用，Agent 也走队列；设置 max_iterations=5 限制工具调用次数 |
| 5 | 中文 Embedding 效果不理想 | 低 | RAG 检索不准确 | 可切换为更大的模型（m3e-base），或调整 similarity 阈值 |
| 6 | Agent 幻觉（调用不该调用的工具） | 中 | 游戏逻辑出错 | Tool 描述写清楚、prompt 中约束工具使用场景、设置 max_iterations 上限 |
| 7 | 降级链路复杂度增加 | 中 | 故障排查困难 | 保留原有 puzzle_bank 降级作为最终 fallback，日志中记录每级降级原因 |
