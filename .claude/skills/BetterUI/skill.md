# AI Frontend Vibe Coding Rulebook

## Core Philosophy

- Minimal over flashy
- Typography first
- Whitespace is premium
- Consistency > creativity
- Motion should support UX, not distract
- Fewer colors, better hierarchy

---

# Visual Style

## Preferred References

- Linear
- Vercel
- Stripe
- Apple
- Raycast
- Notion

## Avoid

- Excessive gradients
- Random colors
- Heavy shadows
- Glassmorphism abuse
- Over-animated UI
- Dense layouts
- Inconsistent spacing

---

# Layout Rules

## Spacing System

Use 8pt spacing system only:

- 4px
- 8px
- 16px
- 24px
- 32px
- 48px
- 64px

Never use random spacing values.

---

# Typography Rules

## Principles

- Large headings
- Comfortable line height
- Minimal font weights
- Strong visual hierarchy

## Preferred Font Style

- Clean sans-serif
- Typography-focused layout
- Generous whitespace

---

# Color Rules

## Palette

- Neutral gray based
- 1 primary color
- 1 accent color max

## Requirements

- Low saturation
- Avoid rainbow palettes
- Maintain strong contrast
- Prefer subtle UI colors

---

# Component Rules

## Border Radius

Use consistent radius:

- cards: 16px
- buttons: 12px
- modals: 20px

## Shadows

- Soft and subtle only
- Never heavy floating shadows

## Cards

- Minimal borders
- Clean separation
- Consistent padding

---

# Motion Rules

## Animation Philosophy

- Smooth
- Fast
- Purposeful
- Minimal

## Preferred Effects

- Fade
- Slide
- Subtle scale
- Soft hover transitions

## Avoid

- Bounce everywhere
- Overlong transitions
- Distracting motion

---

# Tech Stack

## Preferred Stack

- Next.js
- Tailwind CSS
- shadcn/ui
- Framer Motion

## Architecture

- Component-first
- Reusable layouts
- Consistent design tokens
- Responsive by default

---

# AI Coding Rules

## When Generating UI

Always:

- prioritize spacing
- prioritize typography
- maintain consistency
- reduce clutter
- simplify hierarchy

Never:

- generate random styles
- mix multiple visual systems
- overuse gradients
- create crowded layouts

---

# Dashboard Design Rules

## Dashboard Philosophy

Enterprise-grade minimalism.

## Requirements

- Clear hierarchy
- Spacious cards
- Strong typography
- Muted palette
- Clean data presentation

---

# Landing Page Rules

## Hero Section

- One strong headline
- One clear CTA
- Minimal supporting text
- Visual focus above fold

## Content Sections

- Alternating rhythm
- Large spacing
- Consistent card system
